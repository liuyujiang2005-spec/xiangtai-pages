# 主站「还是没有评估码」修复说明

## 原因

- **客服页**生成的「演示码」会存到浏览器 `localStorage` 的 `xiangtai_demo_codes`。
- 若主站用的是 **xiangtai-index.html**，它目前只认固定 5 个码，**没有**读取 `xiangtai_demo_codes`，所以客服生成的码在主站会提示错误。

## 解决办法

### 方式一：用部署站（推荐）

直接使用**同一域名**下的主站 + 客服页，演示码即可通用：

- 主站：https://reliable-empanada-4e952f.netlify.app/
- 客服：https://reliable-empanada-4e952f.netlify.app/admin-eval.html

在客服页点 **「快速生成（演示码）」**，生成后复制「链接+全部码」发给客户，客户打开上面主站链接输入评估码即可。

---

### 方式二：修改 xiangtai-index.html（本地/自建站用）

在 **xiangtai-index.html** 里做 3 处替换（与 xiangtai-pages/index.html、xiangtai-deploy/index.html 一致）：

**1）把固定码改为可合并演示码**

找到：

```javascript
      var VALID_CODES=['XT2024A001','XT2024A002','XT2024A003','DEMO001','DEMO002'];
      var usedCodes=JSON.parse(localStorage.getItem('xiangtai_used_codes')||'[]');
```

替换为：

```javascript
      var DEMO_CODES=['XT2024A001','XT2024A002','XT2024A003','DEMO001','DEMO002'];
      function getDemoCodeList(){ var list=DEMO_CODES.slice(); try{ var stored=JSON.parse(localStorage.getItem('xiangtai_demo_codes')||'[]'); stored.forEach(function(c){ if(list.indexOf(c)===-1) list.push(c); }); }catch(e){} return list; }
      var usedCodes=JSON.parse(localStorage.getItem('xiangtai_used_codes')||'[]');
```

**2）校验时用 getDemoCodeList()**

找到：

```javascript
        if(VALID_CODES.indexOf(code)===-1){setErr('评估码错误，请核对后重试或联系客服');setUnlocked(false);return;}
```

替换为：

```javascript
        if(getDemoCodeList().indexOf(code)===-1){setErr('评估码错误，请核对后重试或联系客服');setUnlocked(false);return;}
```

**3）提交时用 getDemoCodeList()**

找到：

```javascript
        if(VALID_CODES.indexOf(code)===-1){setErr('评估码错误，无法提交');return;}
```

替换为：

```javascript
        if(getDemoCodeList().indexOf(code)===-1){setErr('评估码错误，无法提交');return;}
```

保存后，主站就会认可客服页「快速生成（演示码）」生成的码（需同一域名或同一浏览器本地测试）。
