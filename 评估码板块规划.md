# 评估码板块规划

基于「取消插件、后台手动/一键生成、复制后自动删除、输入匹配后跳转评估界面」等需求，对评估码板块做统一规划。

---

## 一、板块定位与目标

| 项目 | 说明 |
|------|------|
| **定位** | 方案评估的**通行证**：区分意向客户与随意访问，仅持码客户可进入评估表。 |
| **目标** | 意向客户通过微信等联系获取评估码 → 在主站输入正确评估码 → 匹配通过后跳转至评估界面 → 一码一次，用完即废。 |
| **约束** | 不接入任何外部插件或第三方服务，全部在后台与主站本地完成。 |

---

## 二、方案评估码机制（核心功能）

1. **意向客户通过微信联系获取评估码**  
   客服在后台工作台生成或手动添加评估码，将「评估页链接 + 评估码」复制后通过微信等发给客户。

2. **输入正确评估码才能进入方案评估界面**  
   主站「评估方案」入口仅展示输入框；客户输入评估码后，系统校验格式与是否已使用，**仅匹配通过后**才展示评估表单并滚动到该区域（跳转评估界面）。

3. **评估码一次性使用，用完即失效**  
   同一评估码在主站仅能通过校验并提交一次；提交后写入已使用记录，该码再次输入将提示「已使用」。

4. **防止同行无限制查看方案**  
   无有效评估码无法看到评估表内容，评估码由后台控制发放，且复制后后台列表自动清空，降低重复发放与泄露风险。

---

## 三、后台管理系统（客服工作台）

| 功能 | 说明 |
|------|------|
| **内部登录** | 打开后台需先输入账号、密码（默认 `admin` / `Xiangtai2024`），保障系统安全；支持退出登录。 |
| **手动添加评估码** | 在「手动添加评估码」输入框中输入符合格式的码（XT+日期+6位字符），点击「添加」加入当前列表，可与一键生成的码一并复制发放。 |
| **一键生成** | 选择数量（1～10），点击「一键生成」，在本地即时生成评估码，无需插件或外网请求。 |
| **一键复制** | 支持「复制全部通行证」「复制链接+全部码（发给客户）」及单条复制。 |
| **复制成功后自动删除** | 任一次复制成功后，本页已显示的评估码区域自动清空，确保一次性使用、避免重复发放。 |

**涉及页面**：`admin-eval.html`（与主站同域名部署）。

---

## 四、主站评估入口与匹配跳转

| 环节 | 行为 |
|------|------|
| **入口展示** | 「方案评估」区块先只显示：说明文案 + 评估码输入框 + 错误提示区；评估表单区域默认隐藏。 |
| **输入与校验** | 客户输入评估码（支持输入时/失焦/回车触发校验）。校验逻辑：是否已使用 → 格式与签名是否合法。 |
| **匹配通过** | 校验通过后：显示评估表单区域，并**滚动到该区域**（跳转评估界面），客户可填写并提交。 |
| **提交与核销** | 提交时再次校验并标记该码已使用，写入本地已用列表；同一码再次输入提示已使用。 |

**涉及页面**：主站 `index.html` 中 `#evaluate` 区块及 `eval-pass.js` 校验逻辑。

---

## 五、技术实现要点

| 项目 | 实现方式 |
|------|----------|
| **无插件** | 不接入 Supabase 等第三方；不依赖额外 CDN 脚本（仅共用 `eval-pass.js`）。 |
| **评估码格式** | `XT` + 日期(8位) + 6位字符（含签名），由 `eval-pass.js` 统一生成与校验。 |
| **一码一次** | 主站用 `localStorage` 的 `xiangtai_used_codes` 记录已使用码，校验时先查该列表。 |
| **匹配后跳转** | 校验通过后对评估表单容器执行 `scrollIntoView({ behavior: 'smooth', block: 'start' })`。 |
| **后台登录** | 账号/密码与 `eval-pass.js` 内 `ADMIN_USER` / `ADMIN_PASS` 比对，通过后写 `sessionStorage`，刷新需重新登录。 |

## 六、文件与部署清单

| 文件 | 作用 |
|------|------|
| **eval-pass.js** | 评估码生成（`generate`）、校验（`verify`）、后台账号密码常量；主站与后台共用一个文件。 |
| **index.html** | 主站；含评估码输入区、校验逻辑、评估表单区及匹配后跳转。 |
| **admin-eval.html** | 客服工作台；登录、手动添加、一键生成、复制及复制后自动清空。 |
| **评估码配置说明.md** | 使用与配置说明（登录、格式、部署注意）。 |
| **评估码板块规划.md** | 本规划文档。 |

部署时需将上述前端文件置于**同一域名**下；无需数据库或后端服务。

---

## 七、流程简图

```
[客服] 登录后台 → 手动添加 或 一键生成 评估码 → 复制（链接+码）→ 发给客户
                    ↓
[复制成功] 后台本页评估码列表自动清空
                    ↓
[客户] 打开链接 → 在「方案评估」输入评估码 → 匹配通过 → 自动跳转至评估表单 → 填写提交
                    ↓
[该码] 标记已使用，再次输入提示已使用
```

以上为评估码板块的完整规划，与当前实现一致，可直接作为需求与验收依据。
