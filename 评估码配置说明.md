# 方案评估通行证 - 说明

> 完整板块规划见 **评估码板块规划.md**。

**通行证** = 客户在执行阶段拿到的**一次性密码**，仅匹配通过后才能进入评估表，用完即废。**无需对接任何外部服务**，本地自动生成、主站校验。

---

## 一、流程说明

1. **客服**：打开 **admin-eval.html** → 选择生成数量 → 点击「生成通行证」→ 复制「链接+全部码」发给客户。
2. **客户**：打开链接，在「评估方案」入口输入通行证 → **仅当通行证正确且未使用时**才会显示下一界面（评估表）→ 填写并提交后，该码即作废（一码一次）。

---

## 二、后台管理（客服工作台，无插件）

- **登录**：打开 admin-eval.html 需先输入内部账号与密码（默认账号 `admin`，密码 `Xiangtai2024`，可在 eval-pass.js 中修改）。
- **手动添加评估码**：在「手动添加评估码」输入框中输入符合格式的评估码（XT+日期+6位字符），点击「添加」即可加入当前列表，可与一键生成的码一起复制发放。
- **一键生成**：选择数量后点击「一键生成」，本地即时生成评估码，无需任何插件或外部服务。
- **复制后自动删除**：任一次复制成功后，本页已显示的评估码会自动清空，确保一次性使用、防止重复发放。

## 三、评估方案上传（可选）

- 若在 **supabase-config.js** 中填写了 Supabase 的 URL 和 anon key，客户提交评估表后，数据会写入 Supabase 的 **eval_submissions** 表。
- 在 Supabase 后台执行 **supabase-eval-submissions.sql** 可创建该表；客服可在 Table Editor 中查看所有提交记录。
- 未配置 Supabase 时，提交仍会正常核销评估码并在页面展示结果，只是不会上传到云端。

## 四、技术说明

- **生成**：客服页使用 **eval-pass.js** 在本地生成带签名的通行证（格式：XT+日期+6位字符），不依赖 Supabase 或任何插件。
- **校验**：主站使用同一 **eval-pass.js** 校验签名，并配合 **localStorage** 记录已使用码，实现一码一次。
- **部署**：需将 **eval-pass.js**、**index.html**、**admin-eval.html** 放在同域名下；如需上传评估方案，另需配置 Supabase 并执行 supabase-eval-submissions.sql。

## 五、若曾使用 Supabase（可选）

本方案已改为**纯本地通行证**，不再依赖 Supabase。若之前配置过 Supabase，可忽略；保留 **supabase-config.js** 或 **supabase-eval-codes.sql** 不影响当前通行证逻辑。
